| المحور | الحالة قبل | الحالة بعد هذا الـ PR |
|---|---|---|
| الاختبارات | اختبار واحد افتراضي **فاشل** (عدّاد غير موجود) | **28 اختبارًا ناجحًا** (ثيمات، Provider، نماذج) |
| تحليل الكود | 0 errors، 6 warnings، 380+ info (على 3.44) | **0 errors، 0 warnings** على 3.24.5 |
| ثيمات الألوان | 3 ثيمات (كلاسيكي/ليلي/وردي) | **6 ثيمات** + منتقي مرئي (إضافة: أخضر مريح، ذهبي، بيج) |

## 3. التقييم المعماري — نقاط القوة والضعف

### 3.1 نقاط القوة
- فصل طبقات واضح وأسماء معبّرة بالعربية، يسهّل الفهم.
- استخدام `Provider` لإدارة الحالة بشكل متّسق عبر `MultiProvider`.
- طبقة بيانات محلية (`assets/data/*.json`) تجعل التطبيق يعمل دون إنترنت.
- دعم RTL/LTR و5 لغات، وبنية l10n قياسية عبر ملفات `.arb`.

### 3.2 نقاط الضعف (مرتّبة حسب الأثر)

1. **مصدرا حقيقة للثيم (Single Source of Truth مفقود).**
   يوجد تعريف الألوان في مكانين: `ThemeColors` (المستخدم فعليًا في `AppTheme`) و`AppColors.getPrimaryColor/Secondary` (مفاتيح نصية). أي ثيم جديد يتطلب التحديث في عدة أماكن، ما يزيد احتمال التضارب.
   - *عولج جزئيًا:* الثيمات الجديدة أُضيفت في الموضعين، والاختبارات تحرس التطابق. *يبقى:* توحيد المصدر في `ThemeColors` وجعل `AppColors` يفوّض إليه.

2. **الثيم لا يَسري على كامل الواجهة (الأخطر بصريًا).**
   **28 ملفًا** في `screens/` و`widgets/` تستخدم `AppColors.primaryGreen` / `primaryGold` مباشرةً بدلًا من ألوان الثيم النشط أو `Theme.of(context).colorScheme`. النتيجة: تبديل الثيم (أخضر/ذهبي/بيج/ليلي/وردي) يغيّر شريط التطبيق والأزرار فقط، بينما تبقى البطاقات والأيقونات خضراء/ذهبية.
   - *التوصية:* استبدال الألوان الثابتة بـ `Theme.of(context).colorScheme.primary/secondary` أو `ThemeColors(themeProvider.colorTheme)`.

3. **الخدمات Static (يصعب اختبارها وحقنها).**
   `VerseRepository`, `NotificationService`, `LocalStorageService` تعتمد دوالًا ثابتة (`static`) وحالة عامة. هذا يمنع الـ mocking ويجعل اختبارات الوحدة للمنطق الأعلى صعبة.
   - *التوصية:* تحويلها إلى كائنات تُحقن عبر `Provider`/الباني (Constructor Injection) لتسهيل الاختبار.

4. **شيفرة ميتة وتحذيرات.** دوال غير مستخدمة (`_buildHeader`، `_buildQuickStats`، `_buildExploreCard`) ومتغيّرات/حقول غير مستخدمة وفحوص null زائدة. *عولج جزئيًا في هذا الـ PR.*

5. **ترجمات ناقصة.** `flutter gen-l10n` يُبلّغ عن **67 رسالة غير مترجمة** لكل من fr/id/ur — تظهر للمستخدم بالعربية (لغة القالب) بدل لغته.

6. **لا يوجد CI.** لا تكامل مستمر يمنع تكرار «الاختبار الفاشل» أو ظهور أخطاء تحليل.

7. **ديون توافق الإصدارات.** الاعتماد على واجهات تتغيّر بين إصدارات Flutter (مثل `CardTheme`) دون اختبار CI متعدد الإصدارات.

## 4. ما تم تنفيذه فعليًا في هذا الـ PR

1. **3 ثيمات ألوان جديدة** مريحة للعين تُضاف إلى الثلاثة الحالية:
   - **الأخضر المريح (Sage)** — أخضر هادئ منخفض التشبع.
   - **الذهبي (Golden)** — ذهبي دافئ مع لمسة بنية.
   - **البيج (Beige/Sepia)** — رملي دافئ محايد، الأنسب لراحة العين والقراءة الطويلة.
2. **منتقي ثيمات مرئي احترافي:** بدل قائمة الراديو النصية، أصبح كل ثيم يُعرض بدائرتي معاينة (أساسي/ثانوي) + مؤشر اختيار، مع «معاينة لونية مصغّرة» بجانب الإعداد نفسه.
3. **توطين كامل** لأسماء/أوصاف الثيمات الجديدة في اللغات الخمس.
4. **بدائل احتياطية (fallbacks)** لأسماء الثيمات تعتمد على امتداد الـ enum، فلا تظهر قيم فارغة لو غاب مفتاح ترجمة.
5. **مجموعة اختبارات حقيقية (28 اختبارًا):** لوحات الألوان لكل ثيم، تمايز الألوان، بناء `ThemeData` فاتح/داكن، سلوك `ThemeProvider` مع تخزين مُحاكى، و(de)serialization لـ `VerseModel`.
6. **تنظيف التحذيرات** + توسيع قيد `intl` لتوافق أوسع.

> **الحفاظ على التوافق الرجعي:** أُضيفت الثيمات في نهاية الـ enum للحفاظ على فهارس التفضيلات المحفوظة (classic=0…)، مع اختبار يحرس هذا الترتيب.

## 5. خطة إعادة الهيكلة (مراحل، مع المفاضلات)

### المرحلة 0 — تثبيت الجودة (أساس لكل ما بعده)
- إضافة **GitHub Actions**: `flutter analyze` + `flutter test` على كل PR (مصفوفة إصدارات 3.24 والأحدث).
- تفعيل قواعد lint أكثر صرامة تدريجيًا في `analysis_options.yaml`.
- *الفشل المحتمل:* كسر البناء على إصدار أحدث ← يُكتشف مبكرًا عبر مصفوفة الإصدارات.

### المرحلة 1 — توحيد نظام الثيم (أعلى أثر بصري)
- جعل `ThemeColors` المصدر الوحيد، وتفويض `AppColors` إليه.
- استبدال الألوان الثابتة في الـ 28 ملفًا بـ `colorScheme`/`ThemeColors`.
- **3 مسارات ممكنة:**
  1. *استبدال يدوي تدريجي شاشة بشاشة* — أأمن، يسمح بمراجعة بصرية لكل شاشة (موصى به).
  2. *Extension على `BuildContext`* مثل `context.brand.primary` لتقليل اللفظ المتكرر.
  3. *إعادة كتابة شاملة دفعة واحدة* — أسرع لكنه عالي المخاطرة بصريًا.
  - **التوصية:** المسار (1)+(2): إضافة Extension ثم الترحيل تدريجيًا، مع لقطات شاشة (golden tests) لكل ثيم.

### المرحلة 2 — قابلية الاختبار (حقن الاعتماديات)
- تحويل الخدمات الـ static إلى كائنات تُحقن، وكتابة اختبارات لـ `VerseRepository`/منطق «آية اليوم».

### المرحلة 3 — اكتمال الترجمة والوصولية
- ترجمة الـ 67 مفتاحًا الناقصة، وإضافة `untranslated-messages-file` لرصد النواقص.
- فحص تباين الألوان (WCAG AA) لكل ثيم في الوضعين.

## 6. تحسينات تجربة المستخدم والتصميم (UX/UI)

- **معاينة فورية للثيم:** عند فتح المنتقي، عرض بطاقة آية مصغّرة تتلوّن مباشرة بالثيم المختار قبل التأكيد.
- **اتباع ثيم النظام:** خيار «تلقائي» (`ThemeMode.system`) إلى جانب الفاتح/الداكن.
- **وضع القراءة المريح:** سطوع/تباين أهدأ + خلفية بيج (سيبيا) للجلسات الطويلة — الثيم الجديد «البيج» يخدم هذا مباشرة.
- **تكبير الخط الحي:** معاينة حجم الخط على نص آية فعلي داخل نافذة الاختيار.
- **اتساق المسافات والحواف:** توحيد `borderRadius`/الـ paddings عبر ثوابت تصميم (Design Tokens).
- **حالات فارغة وأخطاء أوضح:** رسائل موحّدة عبر `EmptyState` مع إجراء مقترح (CTA).
- **Haptics وانتقالات Hero** عند فتح تفاصيل الآية لإحساس أرقى.
- **إتاحة (Accessibility):** Semantics labels للأيقونات، واحترام `MediaQuery.textScaler` بدل `textScaleFactor` المهمل.
